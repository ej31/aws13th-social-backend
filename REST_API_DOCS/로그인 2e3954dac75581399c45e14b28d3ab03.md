# 로그인

리소스: Auth(인증)
method: POST
API path: auth, login
Error: 400, 401, 404, 415, 422
비고: 이메일과 비밀번호를 통해 인증 후 토큰을 발급합니다.

### 로그인

**POST**`/auth/login`

이메일과 비밀번호를 통해 인증 후 토큰을 발급합니다.

**Request Headers**

| 헤더 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| Content-Type | application/json | ✅ | 요청 바디를 JSON 형식으로 전송하기 위한 타입 |

**Request Body**

| 필드 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| email | string | ✅ | users에 등록된 이메일이어함 |
| password | string | ✅ | 가입 시 설정한 비밀번호와 일치해야 함 |

**Request Example** 

```json
{
  "email": "123psk@naver.com",
  "password": "p@sswordex!"
}
```

**Response 200**

```json
{
  "status": "success",
  "data": {
    "id" : "user_1234567abc",
    "email" : "sigwan116@gmail.com",
    "nickname" : "sihwa",
    "profile_image_url" : "https://api.example.com/uploads/profiles/default.png",
    "access_token" : "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJzdWIiOiJ1c2VyXzdmM2EyYzkxIiwiaWF0IjoxNzM2MjQxNjAwLCJleHAiOjE3MzYyNDUyMDB9.signature_example",
    "token_type" : "Bearer"
  }
}
```

**Error**

| HTTP | code | 언제 나옴 |
| --- | --- | --- |
| 400 | COMMON_400_01 | JSON 파싱 실패 |
| 401 | AUTH_401_04 | 이메일/비밀번호 불일치(Invalid credentials) |
| 404 | USERS_404_01 | 사용자를 찾을 수 없음(정책에 따라 401로 통일해도 됨) |
| 415 | COMMON_415_01 | Content-Type이 application/json이 아님 |
| 422 | AUTH_422_01 | email 형식 오류 |
| 422 | AUTH_422_02 | password 누락/형식 오류 |
