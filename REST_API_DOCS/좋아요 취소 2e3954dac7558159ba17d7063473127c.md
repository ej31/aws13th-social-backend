# 좋아요 취소

리소스: Likes
method: DELETE
API path: likes, posts, {post_id}
Error: 401, 404, 422
비고: 로그인한 사용자가 해당 게시글에 등록한 좋아요를 취소한다.

### 좋아요 취소

**DELETE** `/posts/{post_id}/likes`

로그인한 사용자가 해당 게시글에 등록한 좋아요를 취소한다.

**Request Headers**

| 헤더 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| Authorization | string | ✅ | 좋아요 취소를 하는 사람이 본인임을 확인하기 위한 토큰 (`Bearer <access_token>` 형식의 인증 토큰)(예: `Authorization: Bearer eyJ...`) |

**Path Parameters**

| 파라미터 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| `post_id` | string | ✅ | 좋아요가 속한 게시글의 고유 ID |

**Request Example** 

```
DELETE /posts/post_01/likes
```

**Response (204 No Content)**

```
(no response body)
```

**Error**

| HTTP | code | message | 발생 조건 |
| --- | --- | --- | --- |
| 401 | AUTH_401_01 | Authorization required. | `Authorization` 헤더 없음 |
| 401 | AUTH_401_02 | Invalid token. | 토큰 위조/형식 오류 |
| 401 | AUTH_401_03 | Token expired. | 토큰 만료 |
| 404 | POSTS_404_01 | Post not found. | 해당 `post_id` 게시글 없음 |
| 404 | LIKES_404_01 | Like not found. | 좋아요가 없는 상태(이미 취소됨 포함) |
| 422 | POSTS_422_07 | Invalid post_id. | `post_id` 형식 오류 |
